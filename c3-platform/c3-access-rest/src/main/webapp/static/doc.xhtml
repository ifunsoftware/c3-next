1. Filesystem

Description

This endpoint defines number of method to work with virtual filesystem implemented on top of C3 resource storage

Resource URI

http://c3.ifunsoftware.com/rest/fs/{file-path}
file-path
Absolute file path separated with /

1.1 Create a file or directory

Description

Create a new file or directory in the C3 system. Request URL must point on the directory node of the file system

URI

http://c3.ifunsoftware.com/rest/fs/{file-path}

URI Parameters

file-path
Absolute file path separated with /

HTTP Method

POST

Resource Formats

  • multipart/form-data
  • varying

Authentication

not required

Request Entities

                                x-c3-nodetype
Option HTTP header that indicates type of the created node. By default it is 'file'. Possible values 'file' and 'directory' If request type is multipart/form-data each part with form-field type is treated as user metadata entry and part with type file named 'data' treated as resource content. Otherwise all request body is treated as resource data. Metadata may be set by adding 'x-c3-metadata' headers with metadata contents in the format 'key:Base64(value)'. This is applicable only to file creation 
                                x-c3-type
Optional HTTP header indicating expected result type. Supported values: 'xml', 'json'. Default - 'xml'

Responses

201 Created
Resource created. Response contains new resource address
403 Forbidden
Specified authentication token is not valid

Response Entities

                                <?xml version="1.0" encoding="UTF-8"?>
                                            <p:response xmlns:p="http://c3.aphreet.org/rest/1.0" xsi:schemaLocation="http://c3.aphreet.org/rest/1.0 http://c3-system.googlecode.com/files/rest.xsd" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">

3.6 Update a resource

Description

Create a new resource in the C3 system

URI

http://c3.ifunsoftware.com/rest/resource/{resource-address}

URI Parameters

resource-address
Unique resource identifier inside C3 system

HTTP Method

PUT

Resource Formats

  • multipart/form-data
  • varying

Authentication

not required

Request Entities

If request type is multipart/form-data each part with form-field type is treated as user metadata entry and part with type file named 'data' treated as resource content. Otherwise all request body is treated as resource data. Metadata may be set by adding 'x-c3-metadata' headers with metadata contents in the format 'key:Base64(value)' 
                                x-c3-type
Optional HTTP header indicating expected result type. Supported values: 'xml', 'json'. Default - 'xml'
